Structural Hazards

When a machine is pipelined, the overlapped execution of instructions requires duplication of resources accessed by these instruction stages to allow all possible combinations of instructions in the pipeline. If some combination of instructions cannot be accommodated because of a resource conflict, the machine is said to have a structural hazard.

Common instances of structural hazards arise when some resource has not been duplicated enough to allow all combinations of instructions in the pipeline to execute.

Below is an example where multiple instructions try to use the memory during the same cycle:

load
instr 1
instr 2
instr 3

The above causes a hazard so we will fix this by stalling the pipeline for a cycle

load
instr 1
instr 2
stall
instr 3

Stalling reduces the CPI, so by having multiple instances of the memory resource, both instructions can access memory in the same cycle as seen in the MIPS control path with hazards.
